Work Projects

Data Automation ETL

Automation of data flows into our PostGIS enabled PostgreSQL database is a frequent project I'm part of. This includes building new ETL pipelines and maintaining old ones. We use these to streamline the collection, processing, and integration of disparate datasets into our PostgreSQL database, ensuring the data remains accurate and up-to-date. Python is used to obtain both freely available data or data accessed via paid for credentials. We often pull that data into a GeoPandas DataFrame for processing before loading it into our database. From there we are able to access this spatial data using QGIS or ArcGIS Pro to perform Geospatial Analysis and stylize it before creating the connection to Geoserver. Automation is setup through the use of Cron Jobs in Linux on our virtual servers. ETL pipelines have been created for data sources such as NASA, Fish and Wildlife, NOAA, CalFire, Property Data for all 58 California counties, and more.

Metrics Pipeline and Reporting

I was tasked with establishing a comprehensive Metrics Pipeline and Reporting system to gain insights into user interactions with our web maps. To achieve this, I designed and implemented a user logging schema within our PostgreSQL database. The project seamlessly integrated SQL, Python, Pandas, NumPy, and Matplotlib to analyze data, generate meaningful metrics, and present them through visually informative graphs.

Technical Documentation

I led the creation of technical documentation including an overhaul of our user activity logging database schema. Original activity logging had over 22,000 unique event names in the database. The addition of JSONB added details not previously included in our activity logging.
